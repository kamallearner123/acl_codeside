The Fragmentation Problem

Traditional systems debugging is plagued by tool fragmentation and memory overload. To get a complete view of a machine's health, a developer must parse the output of dozens of low-level tools with inconsistent flags.

top vmstat lsof strace nvidia-smi

The Solution

DevInspect unifies diagnostics. It collects, normalizes, and validates system environments, hardware capabilities, and modern development stacks in a single, secure binary.

devinspect ~ diagnostics

kamal@ubuntu:~$ devinspect doctor

Core Commands

devinspect summary
Unified high-level overview of OS, CPU, memory, GPU, and network.
devinspect hardware
Detailed physical hardware diagnostics including BIOS and sensors.
devinspect network
Comprehensive networking snapshot including MTU, open ports, and latency.
devinspect usb
Connected USB devices structured as a hierarchical bus tree.
devinspect top -t 5
Monitors top CPU/memory consuming processes with continuous sampling.
devinspect pidstat -p <PID>
Surgical inspection of a running process (file descriptors, sockets).
devinspect ai
Validates PyTorch, TensorFlow, CUDA, TensorRT, and local LLM services.
devinspect doctor
Runs sanity checks across compilers and drivers with remediation steps.
AI-First API

Structured for AI Agents

DevInspect is built with AI automation in mind. By bypassing raw console commands, agents can retrieve structured, clean, and normalized inputs in JSON or YAML formats directly inside their execution environments.

  • LLM Context Bootstrapping Use --llm to generate high-density Markdown optimized for LLM token efficiency.
  • Deterministic Parsing Stop writing fragile regex parsers. Use --json for robust JSON API outputs.
